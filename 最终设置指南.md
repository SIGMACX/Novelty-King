## ✅ 好消息

我已经简化了投稿系统，**不再需要配置 Supabase Storage**！

## 📝 设置步骤

### 第一步：登录 Supabase

1. 访问 https://supabase.com
2. 使用您的 Google 账户登录
3. 在项目列表中找到并点击您的项目

### 第二步：执行数据库脚本

1. 在 Supabase 控制台，点击左侧的 **SQL Editor**
2. 点击 **New Query**
3. 复制 `simple-setup.sql` 文件的全部内容（下方有完整脚本）
4. 粘贴到 SQL Editor 中
5. 点击右下角的 **Run** 按钮

### 第三步：测试投稿

1. 访问 http://localhost:3001
2. 登录您的账户
3. 点击 **Submit / 投稿**
4. 填写表单并上传 PDF
5. 点击 **提交论文**

✅ 应该会看到："投稿成功！我们将在3个工作日内完成初审。"

## 📋 完整 SQL 脚本

```sql
-- 简化版设置脚本（不需要 Storage）

-- 1. 创建投稿表
CREATE TABLE IF NOT EXISTS submissions (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  first_author TEXT NOT NULL,
  corresponding_author TEXT,
  email TEXT NOT NULL,
  research_field TEXT,
  keywords TEXT,
  abstract TEXT,
  pdf_url TEXT NOT NULL,
  pdf_filename TEXT NOT NULL,
  pdf_size INTEGER NOT NULL,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'under_review', 'accepted', 'rejected')),
  admin_notes TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 2. 创建管理员表
CREATE TABLE IF NOT EXISTS admin_users (
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 3. 创建索引
CREATE INDEX IF NOT EXISTS idx_submissions_user_id ON submissions(user_id);
CREATE INDEX IF NOT EXISTS idx_submissions_status ON submissions(status);
CREATE INDEX IF NOT EXISTS idx_submissions_created_at ON submissions(created_at DESC);

-- 4. 启用行级安全
ALTER TABLE submissions ENABLE ROW LEVEL SECURITY;
ALTER TABLE admin_users ENABLE ROW LEVEL SECURITY;

-- 5. 用户可以查看自己的投稿
CREATE POLICY "Users can view own submissions" ON submissions
  FOR SELECT
  USING (auth.uid() = user_id);

-- 6. 用户可以插入自己的投稿
CREATE POLICY "Users can insert own submissions" ON submissions
  FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- 7. 管理员可以查看所有投稿
CREATE POLICY "Admins can view all submissions" ON submissions
  FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM admin_users
      WHERE admin_users.user_id = auth.uid()
    )
  );

-- 8. 管理员可以更新所有投稿
CREATE POLICY "Admins can update all submissions" ON submissions
  FOR UPDATE
  USING (
    EXISTS (
      SELECT 1 FROM admin_users
      WHERE admin_users.user_id = auth.uid()
    )
  );

-- 9. 管理员可以查看管理员表
CREATE POLICY "Admins can view admin_users" ON admin_users
  FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM admin_users
      WHERE admin_users.user_id = auth.uid()
    )
  );

-- 10. 自动更新时间戳函数
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- 11. 创建触发器
CREATE TRIGGER update_submissions_updated_at
  BEFORE UPDATE ON submissions
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();
```

## 🔧 （可选）设置管理员账户

如果您想访问管理后台 http://localhost:3001/admin：

### 第一步：获取您的用户 ID

在 SQL Editor 中执行：
```sql
SELECT id, email FROM auth.users;
```

找到您的用户 ID（一串 UUID）

### 第二步：添加为管理员

在 SQL Editor 中执行（替换为您的用户 ID）：
```sql
INSERT INTO admin_users (user_id)
VALUES ('您的用户ID');
```

### 第三步：访问管理后台

1. 访问 http://localhost:3001
2. 登录账户
3. 点击右上角用户菜单
4. 应该能看到 **Admin Dashboard / 管理后台** 选项

## ❓ 遇到问题？

### 问题：执行 SQL 时提示策略已存在

**解决**：这是正常的，说明表已经创建过了。可以忽略这些错误。

### 问题：提交时提示"数据库未配置"

**解决**：
1. 检查 SQL 脚本是否成功执行
2. 在 Table Editor 确认 `submissions` 表存在
3. 刷新浏览器页面

### 问题：提交成功但看不到记录

**解决**：在 SQL Editor 中执行查询：
```sql
SELECT * FROM submissions WHERE user_id = auth.uid();
```

如果能看到记录，说明投稿成功，问题可能在前端显示。
