# 管理员登录系统使用指南

## 📚 系统概述

投稿系统现在区分**两种账户类型**：

1. **作者账户** - 用于提交论文、查看投稿状态
2. **管理员账户** - 用于审核论文、发表管理

## 🔐 登录入口区分

### 作者登录
- **访问地址**: http://localhost:3001/login
- **导航按钮**: 页面右上角的 "Author / 作者" 按钮
- **功能权限**:
  - ✅ 提交论文
  - ✅ 查看自己的投稿
  - ✅ 浏览已发表论文
  - ❌ 无法访问管理后台

### 管理员登录
- **访问地址**: http://localhost:3001/admin/login
- **导航按钮**: 页面右上角的 "Admin / 管理" 按钮（红色）
- **功能权限**:
  - ✅ 查看所有投稿
  - ✅ 审核论文状态
  - ✅ 发表/取消发表论文
  - ✅ 添加审核意见
  - ✅ 下载所有 PDF

---

## 🚀 管理员账号设置（首次使用）

### 第一步：注册管理员邮箱账户

有两种方法：

#### 方法 A：通过管理员登录页面注册（推荐）

1. 访问 http://localhost:3001/admin/login
2. 输入您想要设置为管理员的邮箱（例如：admin@example.com）
3. 点击 **发送验证码**
4. 检查邮箱，获取6位验证码
5. 输入验证码，点击 **验证并登录**
6. 此时会提示"您没有管理员权限"（**这是正常的！**继续下一步）

#### 方法 B：通过 Supabase 后台创建

1. 访问 https://supabase.com/dashboard/project/esoswgywjkkijwyvhwdl
2. 点击 **Authentication** → **Users**
3. 点击 **Add user** → **Create new user**
4. 输入邮箱（不需要密码，系统使用 OTP 验证）
5. 点击 **Create user**

---

### 第二步：在 Supabase 中查找用户 ID

1. 访问 https://supabase.com/dashboard/project/esoswgywjkkijwyvhwdl
2. 点击左侧的 **SQL Editor**
3. 点击 **New Query**
4. 输入以下 SQL 查询：

```sql
SELECT id, email, created_at
FROM auth.users
WHERE email = 'admin@example.com';  -- 替换为您的管理员邮箱
```

5. 点击 **Run**
6. **复制** 返回结果中的 `id`（一串类似 `123e4567-e89b-12d3-a456-426614174000` 的 UUID）

---

### 第三步：添加到管理员表

在同一个 SQL Editor 中：

1. 输入以下 SQL（将 `您的用户ID` 替换为第二步复制的 UUID）：

```sql
INSERT INTO admin_users (user_id)
VALUES ('您的用户ID');
```

**示例**：
```sql
INSERT INTO admin_users (user_id)
VALUES ('123e4567-e89b-12d3-a456-426614174000');
```

2. 点击 **Run** 执行

---

### 第四步：验证设置成功

执行以下 SQL 查询，确认管理员已添加：

```sql
SELECT
  au.user_id,
  u.email,
  au.created_at as admin_since
FROM admin_users au
JOIN auth.users u ON au.user_id = u.id;
```

应该能看到刚才添加的管理员邮箱。

---

### 第五步：测试管理员登录

1. 访问 http://localhost:3001/admin/login
2. 输入管理员邮箱
3. 点击 **发送验证码**
4. 检查邮箱，获取6位验证码
5. 输入验证码，点击 **验证并登录**
6. 应该会提示 **"管理员登录成功！正在跳转..."**
7. 自动跳转到管理后台 http://localhost:3001/admin

✅ **完成！** 您已成功设置管理员账户。

---

## 📖 管理员登录流程

### 每次登录步骤

1. 访问 http://localhost:3001/admin/login
2. 输入管理员邮箱
3. 点击 **发送验证码**
4. 打开邮箱，查找验证码（6位数字）
5. 返回网页，输入验证码
6. 点击 **验证并登录**
7. 自动跳转到管理后台

### 安全说明

- **无需密码**：管理员登录使用邮箱 OTP（一次性密码）验证
- **验证码有效期**：通常为 5-10 分钟
- **自动验证权限**：登录成功后系统会自动检查管理员权限
- **非管理员拒绝**：如果邮箱不在管理员列表中，会自动拒绝登录

## 🎯 管理员功能说明

登录成功后，在管理后台 (http://localhost:3001/admin) 可以：

### 1. 查看所有投稿

- 左侧显示所有投稿列表
- 按提交时间倒序排列
- 显示论文标题、作者、状态

### 2. 审核论文

点击任一投稿，右侧显示详情，可以：

- 查看完整论文信息
- 下载并阅读 PDF
- 更改审核状态：
  - 待审核 → 审核中
  - 审核中 → 已接受 / 已拒绝
  - 已接受 → 已发表

### 3. 发表论文

在论文详情的"发表管理"部分：

- 点击 **🚀 发表论文** 按钮
- 确认发表操作
- 论文将在 http://localhost:3001/publications 公开展示
- 状态自动变为"已发表"

### 4. 取消发表

如需撤下已发表的论文：

- 在"发表管理"部分点击 **取消发表**
- 确认操作
- 论文将不再公开显示

### 5. 添加审核意见

- 在"审核意见"文本框输入反馈
- 点击 **保存审核意见**
- 作者可在"我的投稿"页面查看

## 🔧 管理员账户管理

### 查看所有管理员

在 Supabase SQL Editor 中执行：

```sql
SELECT
  au.user_id,
  u.email,
  au.created_at as admin_since
FROM admin_users au
JOIN auth.users u ON au.user_id = u.id
ORDER BY au.created_at DESC;
```

### 添加新管理员

重复"第一步到第三步"的设置流程。

### 移除管理员权限

在 Supabase SQL Editor 中执行：

```sql
DELETE FROM admin_users
WHERE user_id = '要移除的用户ID';
```

## ❓ 常见问题

### Q1: 登录时提示"您没有管理员权限"

**原因**: 该邮箱账户未添加到管理员表中

**解决**:
1. 确认已执行"第三步：添加到管理员表"
2. 在 SQL Editor 中查询确认：
```sql
SELECT * FROM admin_users WHERE user_id = '您的用户ID';
```
3. 如果没有记录，重新执行 INSERT 语句

---

### Q2: 收不到验证码邮件

**可能原因**:
- 邮件在垃圾箱中
- Supabase 邮件配额已用完
- 邮箱地址错误

**解决**:
1. 检查垃圾邮件文件夹
2. 确认邮箱地址拼写正确
3. 在 Supabase 控制台检查 Authentication → Email Templates
4. 检查 Supabase 项目的邮件发送限额

---

### Q3: 执行 INSERT 时提示 "duplicate key value"

**原因**: 该用户已经是管理员

**解决**: 无需重复添加，直接登录即可

---

### Q4: 执行 INSERT 时提示 "violates foreign key constraint"

**原因**: 用户 ID 不存在

**解决**:
1. 检查第二步的查询结果
2. 确保复制的是正确的用户 ID（UUID 格式）
3. 确认该邮箱账户已在 Supabase Authentication 中创建

---

### Q5: 作者误用管理员登录入口

**行为**: 作者账户尝试通过管理员入口登录

**结果**:
- 如果不在 `admin_users` 表中，会提示"您没有管理员权限"
- 3秒后自动跳转到作者登录页面

**说明**: 这是正常的安全机制，作者应使用作者登录入口

---

### Q6: 管理员也能提交论文吗？

**答案**: 可以，但需要分开操作：

1. **作为管理员**: 通过 http://localhost:3001/admin/login 登录，管理所有投稿
2. **作为作者**: 通过 http://localhost:3001/login 注册/登录，提交自己的论文

管理员账户和作者账户是独立的，一个邮箱可以同时拥有两种身份。

---

## 🔒 安全最佳实践

1. **使用强邮箱**: 管理员邮箱应该使用可靠的邮件服务（如 Gmail、Outlook）
2. **限制管理员数量**: 只为必要的人员分配管理员权限
3. **定期审查**: 定期检查管理员列表，移除不再需要的权限
4. **不共享账户**: 每个管理员应使用独立的邮箱账户
5. **及时登出**: 在公共电脑上使用后，务必登出
